Create a git commit using the following steps:

1. Run `git status` (never use -uall) and `git diff` to review all changes.
2. Use "$ARGUMENTS" as the **first line** of the commit message exactly as provided.
3. Auto-generate a bullet-point body summarizing the key changes. Each bullet should be concise and describe what changed and why.
4. Stage all relevant changed files individually (never use `git add -A` or `git add .`). Do not stage files that contain secrets (.env, credentials) or the .vscode/ directory.
5. Commit with a multiline message in this format:

```
<user's short description>

- <auto-generated bullet>
- <auto-generated bullet>
- ...
```

6. Run `git status` after committing to verify success.

Important:
- Do NOT append Co-Authored-By lines.
- Do NOT push to remote.
- Use a HEREDOC to pass the commit message to git.
