# System 1: The Intake
## The Cognitive Mechanics of Learning

> When engineers fail to learn a new system, it is usually because they overwhelm their working memory by trying to memorize syntax rather than understanding the architecture.

---

## The Psychological Foundations

### Just-in-Time (JIT) Learning
Developers naturally default to **"Just-in-Case" learning** — reading the entire documentation top-to-bottom just in case they need a detail later. This is highly inefficient.

**JIT learning** dictates that you only hunt for the specific information required to overcome your immediate roadblock.

### Bloom's Taxonomy
This educational framework proves you cannot analyze a system until you first remember and understand it. You must move sequentially from lower-order thinking (running the code) to higher-order thinking (evaluating the architecture).

---

## The Expanded Grokking Script

### Step 1 — The JIT Filter
**Bloom's Level: Remember**

> *Setting the Boundary*

**Action:** Before opening a single file, define your exact goal.

**Ask Yourself:** What is the absolute minimum I need to learn today to get this working?

**Output:** A hard boundary that stops you from falling into "Just-in-Case" rabbit holes.

---

### Step 2 — The Macro Scan
**Bloom's Level: Understand**

> *Read the README. Ignore the code.*

**Action:** Read the `README.md` and ignore the code. Identify the core value proposition.

**Ask Yourself:** What specific pain point is this solving?

**Output:** The one-sentence summary of the technology's purpose.

---

### Step 3 — Trace the Happy Path
**Bloom's Level: Apply**

> *Get a successful output. Don't worry about how it works yet.*

**Action:** Execute the "Quickstart" guide to get a successful output. Do not worry about how it works yet.

**Ask Yourself:** What are the minimum inputs required, and what is the exact output?

**Output:** A mapped lifecycle of the user experience.

---

### Step 4 — Functional Decomposition
**Bloom's Level: Analyze**

> *Open the source code. Break it into its core components.*

**Action:** Now, open the source code. Break the system down into its 3–5 core components (e.g., routing, auth, state management).

**Ask Yourself:** How is the "Happy Path" actually implemented under the hood?

**Output:** A granular map of the internal dependencies.

---

### Step 5 — The ELI5 Translation
**Bloom's Level: Evaluate**

> *Explain the entire system using a non-technical analogy.*

**Action:** Force yourself to explain the entire system using a non-technical analogy.

**Ask Yourself:** How would I explain this to a 5-year-old?

**Output:** A solidified mental model that proves you truly grok the concept.

---
